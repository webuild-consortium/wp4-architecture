# What’s Next & Scaling Up
This document and the initial attestations data model and schemas together establishe a common technical baseline for the wallet ecosystem. The architectural design will continue to evolve throughout the project with the main project focus changing towards implementation and testing.

**WP4 - General Capabilities** operates on a set timeline with key milestones and deliverables while adopting in response to feedback from the use cases as well as business, regulatory and technological changes. To this end, the deliverable is expected to go through changes and refinement, as more ADR and CS documents are added or updated. It will serve as a living and dynamically evolving document.
 
The roadmap for Months 8–19 is dedicated to maturing the trust infrastructure and validating cross-border interoperability through the project’s automated testbed.

Along with these deliverables and milestones, WP4 will iterate on these components throughout the project’s duration. All WP4 groups will continuously incorporate feedback from the piloting phase and adapt to new requirements and emerging EUDI standards and other technological developments. The ITB will remain a vital tool for continuous integration and testing, ensuring that the solutions remain interoperable, secure, and scalable.
